# Contributing to NSC Tech Hub Projects

Welcome! To maintain our institutional standards and ensure your work is recognized, all contributors must follow this workflow:

## 🚦 The Rules
1. **No Direct Pushes:** All code must be submitted via a Pull Request (PR).
2. **Issue Linking:** Your PR must link to an existing GitHub Issue created by the TPM. (Example: `Closes #12`).
3. **Vetting Required:** All assets (images/logos) used in your code must have passed the Vetting Loop found in the `documentation` repo.

## 🛠 Branching Strategy
* `main`: Production-ready code only.
* `dev`: Integration testing.
* `feature/[issue-number]`: Your specific task branch.

*Questions? Contact the TPM or your Department Liaison.*
